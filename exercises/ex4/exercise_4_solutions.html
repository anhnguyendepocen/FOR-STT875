Floating point arithmetic

R, as does most software, uses floating point arithmetic, which is not the same as the arithmetic we learn, because computers cannot represent all numbers exactly. This has important consequences when we test for equality of numbers and in other settings we’ll explore in the course.

Here are a few examples.

First, would you expect 0.2 == 0.6/3 to return TRUE or FALSE? Test it out.

# TODO 4.9: Enter 0.2 == 0.6/3 and see what happens. Explain the potentially surprising result in commented text.
0.2 == 0.6/3
[1] FALSE

Next consider the vector point3 created below.

point3 <- c(0.3, 0.4 - 0.1, 0.5 - 0.2, 0.6 - 0.3, 0.7 - 0.4)
point3
[1] 0.3 0.3 0.3 0.3 0.3

Test whether each element of point3 is equal to 0.3

# TODO 4.10: Perform the test here
point3 == 0.3
[1]  TRUE FALSE  TRUE  TRUE FALSE

.Machine is a list that holds information on the numerical characteristics of the machine R is running on, such as the largest double or integer and the machine’s precision. For example

.Machine$double.eps
[1] 2.220446e-16

is the smallest positive floating-point number x such that 1 + x != 1. The false in point3 == 0.3 and 0.2 == 0.6/3 show us that using floating point arithmetic with logical operators can yield unexpected results. The false for 0.2 == 0.6/3 is because floating point arithmetic of 0.6/3 will equal 0.2 + x, where x is some very small value.

If you want to safely apply a logical tests to computed values, R provides a set of functions that test with a given tolerance, e.g., equal up to 1.5e-8 digits. For example,

all.equal(0.2, 0.6/3)
[1] TRUE

We revisit this topic in Chapter 7.3 in the course text.

Here’s another example of the limitations of floating point arithmetic. In calculus we were taught the base of the natural logarithm is \[ e = \lim_{n\to\infty} (1 + 1/n)^n \] So in principle we should be able to get closer and closer approximations to \(e\) by using larger and larger values of \(n\) in the above expression (\(e\) is something close to 2.718282). See here if you need a refresher.

Let’s try a few.

n <- 100
(1 + 1/n)^n
[1] 2.704814
n <- 10000
(1 + 1/n)^n
[1] 2.718146
n <- 1000000
(1 + 1/n)^n
[1] 2.71828
n <- 100000000
(1 + 1/n)^n
[1] 2.718282

So far things are looking good. Investigate what happens as you increase \(n\). Each time multiply \(n\) by 10. Continue until you get to 10000000000000000. What happens?

n <- 1000000000
(1 + 1/n)^n
[1] 2.718282
n <- 10000000000
(1 + 1/n)^n
[1] 2.718282
n <- 100000000000
(1 + 1/n)^n
[1] 2.718282

Things are started going weird beyond this point on my machine (i.e., the approximation is no longer close to \(e\) = 2.71828). The majority computers should start to give bad answers around this size of \(n\).

n <- 1000000000000
(1 + 1/n)^n
[1] 2.718523
n <- 10000000000000
(1 + 1/n)^n
[1] 2.71611
n <- 100000000000000
(1 + 1/n)^n
[1] 2.71611
n <- 1000000000000000
(1 + 1/n)^n
[1] 3.035035
n <- 10000000000000000
(1 + 1/n)^n
[1] 1

You don’t have to understand exactly how your machine handles floating point numbers, but it is good to be aware that testing for equality of numbers may result in unexpected behavior.

Here we will continue using the subset function, learn how to perform basic linear regression in R, and continue developing our plotting skills with the graphics package. We seek to determine if there is a significant linear correlation between year and per capita GDP for the United States. We will do this by subsetting the data frame to obtain only the data from the United States, determine the correlation between year and GDP, and then create a scatter plot of the results. I used the following functions in my answer: subset, cor (correlation), plot, axis (manually add axis to the plot), text, abline (add a line to the plot), lm (linear model, used to get line on plot), expression (way to add mathy text to an R plot, e.g., \(\rho\)). The color I used is lightskyblue1.

us.gm <- subset(gapminder, country == 'United States')
cor(us.gm$year, us.gm$gdpPercap)
[1] 0.9902341
plot(us.gm$year, us.gm$gdpPercap, pch = 24, col = 'black', 
     bg = 'lightskyblue1', axes = FALSE, ylab = 'GDP Per Capita', 
     xlab = 'Year', main = 'United States', xlim = c(1950, 2010), 
     ylim = c(10000, 50000), cex = 1.5)
axis(1)
axis(2)
text(x = 1960, y = 40000, labels = expression(rho~"= 0.99"))
abline(lm(gdpPercap ~ year, data = us.gm), col = 'black', lty = 2)
